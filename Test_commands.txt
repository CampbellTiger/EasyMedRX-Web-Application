  EasyMedRX — Browser Console Test Commands
  Open DevTools with F12, go to the Console tab,
  paste any command below and press Enter.
  You must be logged in as a staff/admin user first.

  IMPORTANT: Paste STEP 0 first every time you open
  DevTools or refresh the page. The testEndpoint()
  helper does not persist between sessions.

────────────────────────────────────────────────────────
  STEP 0 — PASTE THIS FIRST (required every session)
  Paste this helper once per session. It wraps every
  fetch call and prints the HTTP status + raw body
  so you can see exactly what the server returned.
────────────────────────────────────────────────────────

function testEndpoint(url) {
  fetch(url, { credentials: "same-origin" })
    .then(r => {
      console.log("HTTP status:", r.status, r.statusText);
      return r.text();
    })
    .then(t => {
      try { console.log("Response:", JSON.parse(t)); }
      catch { console.log("Raw response (not JSON — likely a login redirect or error page):\n", t.slice(0, 500)); }
    })
    .catch(e => console.error("Fetch failed:", e));
}

────────────────────────────────────────────────────────
  TROUBLESHOOTING
────────────────────────────────────────────────────────

If testEndpoint() prints "not JSON — likely a login redirect":
  → You are not logged in as a staff user.
     Go to /admin/ in the browser, log in, then retry.

If testEndpoint() prints HTTP status 403:
  → Logged in but account is not staff.
     Go to /admin/auth/user/, edit your user,
     tick "Staff status", save, then retry.

If testEndpoint() prints HTTP status 500:
  → Server-side error. Check the terminal running
     Daphne for the Python traceback.

If command 4 shows no pop-up but no error:
  → Notification permission is not granted.
     Run command 1 first and click Allow.

If command 3 shows HTTP 200 but no pop-up:
  → WebSocket is not connected (check command 8)
     or notification permission is denied (command 1).
